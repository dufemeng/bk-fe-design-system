# Surface Fixture: Remove Section

## 用户输入

删掉首页上方那块指标区域，页面节奏重新整理一下。

## 期望行为

- 触发 `bfds-design`。
- 判定 change type 为 `remove` 或 `replace`，取决于是否要补新结构。
- 必须确认删除对象、周边保留范围和空出后的结构处理。
- 必须要求当前页面视觉证据或用户确认。
- 不把“删掉指标区”扩展成新的产品能力。

## Framing 摘要示例

Surface: 首页上方指标区域及其相邻首屏结构  
Current source: 待用户提供截图或浏览器捕获  
Change type: remove  
Keep: 首屏主信息、导航、主要行动入口、品牌 token  
Change: 删除指标区后的垂直节奏、信息密度、过渡区结构  
Avoid: 不新增替代指标，不编造数据，不改后端统计逻辑
