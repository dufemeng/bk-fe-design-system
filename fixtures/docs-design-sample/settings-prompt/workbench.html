方案 A

Quiet Navigation

保留安静导航和低干扰节奏，让提示词输入成为稳定主区域。

DESIGN.md 规则
沿用 restrained 产品 UI、输入控件、按钮和语义状态 token
实现边界
只改输入区层级、帮助信息位置和局部保存反馈
风险
视觉记忆点较少
自审重点
不得改变设置导航、提交逻辑和组件 API

方案 B

Operational Workbench

把输入、模板、预览和操作状态组织成更高效的工作台。

DESIGN.md 规则
沿用面板、分隔、按钮和状态表达规则
实现边界
允许调整输入、模板、预览和状态的相对位置
风险
首次使用信息量更高
自审重点
小屏折叠后输入区仍是第一优先且无文字溢出

方案 C

Explicit Recovery

突出错误恢复和保存反馈，降低配置失败时的操作成本。

DESIGN.md 规则
错误、成功、加载和恢复动作使用语义状态规则
实现边界
增强输入区错误说明、恢复动作和保存反馈
风险
视觉存在感更强，需要控制密度
自审重点
恢复动作不新增后端、权限或未确认配置能力
