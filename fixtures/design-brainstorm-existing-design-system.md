# Design Brainstorm Fixture: Existing Design System

## 输入

项目已有 `DESIGN.md` 和组件 tokens。用户要求重新设计现有输入组件。

## 期望行为

- 先读取 `DESIGN.md`。
- 三个方向落在现有 tokens、组件、icon set 内。
- 如果要突破现有模式，必须说明是 20% 独特选择，并服务产品表达。
- 不另起一套字体、颜色、圆角、图标或框架。

## 方向差异示例

- A：更安静，帮助内容折叠，状态贴近输入。
- B：更高效，历史和模板靠近输入，键盘路径更短。
- C：更强提示，错误和成功状态更显性，但不新增业务规则。
